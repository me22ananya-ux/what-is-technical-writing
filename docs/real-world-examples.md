# Real-World Examples of Technical Writing
Below are some real-world examples of technical writing mentioned that easily demonstrates how different kinds of documentation are used:
1. **Onboarding Guides**
- Onboarding guides are crucial for introducing new users to a product, service, or company. They provide step-by-step instructions which includes setup instructions, key features, and best practices. 
- **For Instance**: A software company may have an onboarding guide that walks users through the process of creating an account, setting preferences, and using key features.

2. **API References**
- API references provide documentation to use the (APIs) of a system or service. It consists of endpoints, data formats, methods, and usage examples.
- **For Instance**: A payment gateway's API reference might document how developers can integrate payment processing into their applications.

3. **Frequently Asked Questions (FAQs)**
- FAQs are a type of technical writing which is used to answer user questions. It helps in finding answers to questions on how to use a product or service.
- **For Instance**: An IT company may have a FAQ page which addresses common troubleshooting steps, like "How do I reset my password?" or "Which browser is safe to login?"
